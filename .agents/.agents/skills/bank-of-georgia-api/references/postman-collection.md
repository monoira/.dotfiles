# Bank of Georgia — Merchant Endpoints Postman Collection Generator

Generate Postman collections that document and test the **merchant's own endpoints** — the webhook handlers, callback receivers, OAuth redirect handlers, and service provider endpoints that the merchant builds on their server. These collections do NOT include BOG's bank-side API URLs.

The purpose: let the merchant (or BOG's integration team) hit the merchant's endpoints to verify they work correctly before go-live.

## Per-Integration Collections

### 1. iPay — Merchant Endpoints (`bog-ipay-merchant.postman_collection.json`)

These are the endpoints the merchant builds to handle BOG iPay webhooks and redirects.

**Webhook Handler**
- `POST {{merchant_base_url}}/webhooks/bog/ipay` — Simulate BOG iPay webhook (payment success)
  - Description: BOG sends this POST to your server when a payment event occurs. Test that your handler processes it correctly.
  - Body (what BOG sends):
  ```json
  {
    "order_id": "{{bog_order_id}}",
    "shop_order_id": "{{bog_shop_order_id}}",
    "event": "order_payment",
    "amount": 100.50,
    "currency": "GEL",
    "status": "success"
  }
  ```
  - Tests:
  ```javascript
  pm.test("Webhook returns 200", function () {
      pm.expect(pm.response.code).to.equal(200);
  });

  pm.test("Response time under 10s (BOG timeout)", function () {
      pm.expect(pm.response.responseTime).to.be.below(10000);
  });
  ```

- `POST {{merchant_base_url}}/webhooks/bog/ipay` — Duplicate webhook (idempotency test)
  - Description: Send the same order_id + event twice to verify idempotent handling.
  - Body: identical to above
  - Tests:
  ```javascript
  pm.test("Duplicate webhook returns 200 (idempotent)", function () {
      pm.expect(pm.response.code).to.equal(200);
  });
  ```

- `POST {{merchant_base_url}}/webhooks/bog/ipay` — Refund event
  - Body:
  ```json
  {
    "order_id": "{{bog_order_id}}",
    "shop_order_id": "{{bog_shop_order_id}}",
    "event": "order_refund",
    "amount": 100.50,
    "currency": "GEL",
    "status": "success"
  }
  ```

**Redirect Endpoints**
- `GET {{merchant_base_url}}/payment/success?order_id={{bog_order_id}}` — Success redirect page
  - Description: Where BOG redirects the customer after successful payment.
  - Tests:
  ```javascript
  pm.test("Success page loads", function () {
      pm.expect(pm.response.code).to.be.oneOf([200, 302]);
  });
  ```

- `GET {{merchant_base_url}}/payment/failed?order_id={{bog_order_id}}` — Fail redirect page
- `GET {{merchant_base_url}}/payment/cancel?order_id={{bog_order_id}}` — Cancel redirect page

**Amount Validation Test**
- `POST {{merchant_base_url}}/webhooks/bog/ipay` — Mismatched amount
  - Description: Send a webhook with a different amount than the original order to verify your handler validates amounts.
  - Body: same structure but with `"amount": 999.99`
  - Tests:
  ```javascript
  pm.test("Handler rejects or flags mismatched amount", function () {
      // Merchant should either reject or flag this for review
      // The exact behavior depends on implementation
      pm.expect(pm.response.code).to.be.oneOf([200, 400, 422]);
  });
  ```

  - Body: `{ "order_id": "{{bog_order_id}}", "shop_order_id": "{{bog_shop_order_id}}", "event": "order_cancelled" }`

**Redirect Endpoints**

- `GET {{merchant_base_url}}/installments/rejected` — Rejected redirect

**Backend API (Merchant's Own)**
- `POST {{merchant_base_url}}/api/bog/create-installment-order` — Merchant's order creation endpoint
  - Description: The merchant's backend endpoint that their frontend calls to initiate an installment. This endpoint then calls BOG's API internally.
  - Body:
  ```json
  {
    "product_id": "PROD-001",
    "amount": 500.00,
    "campaign_id": "CAMPAIGN_ID"
  }
  ```
  - Tests:
  ```javascript
  pm.test("Returns order creation response", function () {
      pm.expect(pm.response.code).to.be.oneOf([200, 201]);
      var json = pm.response.json();
      pm.expect(json).to.have.property("redirect_url");
  });
  ```

### 3. Payment Gateway / Service Provider — Merchant Endpoints (`bog-payment-gateway-merchant.postman_collection.json`)

These are the endpoints the service provider exposes for BOG to call.

**Payment Endpoint**
- `POST {{merchant_base_url}}/api/bog/payment` — Process payment from BOG
  - Description: BOG sends payment requests to this endpoint. Verify it processes correctly.
  - Auth: depends on agreed method (Basic / API Key / HMAC)
  - Body:
  ```json
  {
    "transactionId": "{{bog_transaction_id}}",
    "personalNumber": "{{bog_personal_number}}",
    "amount": 50.00,
    "currency": "GEL"
  }
  ```
  - Tests:
  ```javascript
  pm.test("Payment processed successfully", function () {
      pm.expect(pm.response.code).to.equal(200);
      var json = pm.response.json();
      pm.expect(json.statusCode).to.equal(0);
  });
  ```

- `POST {{merchant_base_url}}/api/bog/payment` — Duplicate transaction (idempotency)
  - Description: Same transactionId sent twice. Should return code `8` (non-unique).
  - Tests:
  ```javascript
  pm.test("Duplicate returns non-unique status", function () {
      var json = pm.response.json();
      pm.expect(json.statusCode).to.equal(8);
  });
  ```

- `POST {{merchant_base_url}}/api/bog/payment` — Invalid customer
  - Body: `{ "transactionId": "TXN-002", "personalNumber": "INVALID", "amount": 50.00, "currency": "GEL" }`
  - Tests: expects statusCode `6` (customer not found)

**Status Check Endpoint**
- `GET {{merchant_base_url}}/api/bog/status?transactionId={{bog_transaction_id}}` — Check transaction status
  - Tests:
  ```javascript
  pm.test("Status check returns valid response", function () {
      pm.expect(pm.response.code).to.equal(200);
      var json = pm.response.json();
      pm.expect(json.statusCode).to.be.oneOf([0, 12, 13, 14]);
  });
  ```

**Cancel Endpoint**
- `GET {{merchant_base_url}}/api/bog/cancel?transactionId={{bog_transaction_id}}` — Cancel transaction
  - Tests: expects statusCode `0` or `12`

**Auth Method Tests**

Include a folder for each supported auth method:

- **Basic Auth**: `Authorization: Basic {base64(client_id:secret)}`
- **API Key**: `Auth-Key: {api_key}` header
- **HMAC-SHA256**: signed request body/query
- **No Auth / Wrong Auth**: should return statusCode `1` (access denied) or `2` (invalid credentials)

### 4. OAuth2 / BOG-ID — Merchant Endpoints (`bog-oauth-merchant.postman_collection.json`)

**OAuth2 Redirect Handler**
- `GET {{merchant_base_url}}/oauth/bog/callback?code={{bog_auth_code}}&state={{bog_state}}` — OAuth2 callback
  - Description: Where BOG redirects after user authorizes. Tests that your handler exchanges the code for a token.
  - Tests:
  ```javascript
  pm.test("OAuth callback processes successfully", function () {
      pm.expect(pm.response.code).to.be.oneOf([200, 302]);
  });
  ```

- `GET {{merchant_base_url}}/oauth/bog/callback?error=access_denied&state={{bog_state}}` — Authorization denied
  - Description: Tests error handling when user denies authorization.
  - Tests:
  ```javascript
  pm.test("Error handled gracefully", function () {
      pm.expect(pm.response.code).to.be.oneOf([200, 302]);
  });
  ```

- `GET {{merchant_base_url}}/oauth/bog/callback?code={{bog_auth_code}}&state=WRONG-STATE` — CSRF protection test
  - Description: Invalid state parameter. Your handler should reject this.
  - Tests:
  ```javascript
  pm.test("Invalid state rejected", function () {
      pm.expect(pm.response.code).to.be.oneOf([400, 403]);
  });
  ```

**PSD2 OAuth2 Redirect (if separate)**
- `GET {{merchant_base_url}}/oauth/bog-psd2/callback?code={{bog_auth_code}}&state={{bog_state}}` — PSD2 OAuth callback

## Test Scripts — Common Patterns

### Webhook handler health
```javascript
pm.test("Handler responds within BOG timeout window (10s)", function () {
    pm.expect(pm.response.responseTime).to.be.below(10000);
});

pm.test("Returns HTTP 200", function () {
    pm.expect(pm.response.code).to.equal(200);
});
```

### Service provider response structure
```javascript
pm.test("Valid payment gateway response", function () {
    var json = pm.response.json();
    pm.expect(json).to.have.property("statusCode");
    pm.expect(json.statusCode).to.be.a("number");
});
```

### Redirect page verification
```javascript
pm.test("Page loads or redirects correctly", function () {
    pm.expect(pm.response.code).to.be.oneOf([200, 301, 302]);
});
```

### Idempotency verification
```javascript
// Run the same request twice and verify consistent behavior
pm.test("Idempotent: same result on repeat", function () {
    pm.expect(pm.response.code).to.equal(200);
});
```
