# TBC Bank — Merchant Endpoints Postman Collection Generator

Generate Postman collections that document and test the **merchant's own endpoints** — the webhook handlers, callback receivers, billing endpoints, and auth flows that the merchant builds on their server. These collections do NOT include TBC's bank-side API URLs.

The purpose: let the merchant (or TBC's integration team) hit the merchant's endpoints to verify they work correctly before go-live.

---

## Per-Integration Collections

### 1. Checkout / TPay — Merchant Endpoints (`tbc-checkout-merchant.postman_collection.json`)

These are the endpoints the merchant builds to handle TBC Checkout callbacks and redirects.

**Webhook / Callback Handler**
- `POST {{merchant_base_url}}/webhooks/tbc/checkout` — Simulate TBC callback
  - Description: TBC sends this POST to your server after payment completes. Test that your handler processes it correctly.
  - Body (what TBC sends):
  ```json
  {
    "PaymentId": "{{tbc_payment_id}}"
  }
  ```
  - Tests:
  ```javascript
  pm.test("Callback returns 200", function () {
      pm.expect(pm.response.code).to.equal(200);
  });

  pm.test("Response time under 5s (TBC timeout)", function () {
      pm.expect(pm.response.responseTime).to.be.below(5000);
  });
  ```

- `POST {{merchant_base_url}}/webhooks/tbc/checkout` — Duplicate callback (idempotency test)
  - Description: Send the same PaymentId twice to verify your handler is idempotent.
  - Body: same as above, identical `PaymentId`
  - Tests:
  ```javascript
  pm.test("Duplicate callback returns 200 (idempotent)", function () {
      pm.expect(pm.response.code).to.equal(200);
  });
  ```

**Redirect Endpoints**
- `GET {{merchant_base_url}}/payment/success?PaymentId={{tbc_payment_id}}` — Success redirect page
  - Description: Where TBC redirects the customer after successful payment.
  - Tests:
  ```javascript
  pm.test("Success page loads", function () {
      pm.expect(pm.response.code).to.be.oneOf([200, 302]);
  });
  ```

- `GET {{merchant_base_url}}/payment/fail?PaymentId={{tbc_payment_id}}` — Fail redirect page
  - Tests: same pattern

**Auth — Merchant's OAuth2 Token Endpoint (if exposed)**
- `POST {{merchant_base_url}}/api/tbc/auth` — Merchant's internal auth proxy (if applicable)
  - Description: Some merchants wrap TBC auth in their own endpoint. Include only if the merchant builds one.

### 2. Installments — Merchant Endpoints (`tbc-installments-merchant.postman_collection.json`)

**Callback Handler**
- `POST {{merchant_base_url}}/webhooks/tbc/installments` — Installment status callback
  - Description: TBC sends status updates here when installment application status changes.
  - Body (simulated TBC callback):
  ```json
  {
    "sessionId": "{{tbc_session_id}}",
    "status": "COMPLETE"
  }
  ```
  - Tests:
  ```javascript
  pm.test("Installment callback returns 200", function () {
      pm.expect(pm.response.code).to.equal(200);
  });
  ```

- `POST {{merchant_base_url}}/webhooks/tbc/installments` — Cancelled status
  - Body: `{ "sessionId": "{{tbc_session_id}}", "status": "CANCELED" }`

**Redirect Endpoints**
- `GET {{merchant_base_url}}/installments/success` — Installment approved redirect
- `GET {{merchant_base_url}}/installments/fail` — Installment rejected redirect

### 3. XML Billing Protocol — Merchant Endpoints (`tbc-billing-merchant.postman_collection.json`)

These are the endpoints that TBC calls on the merchant's (service provider's) server.

**CHECK Command**
- `GET {{merchant_base_url}}/billing/?command=check&account={{billing_account}}` — Validate customer
  - Description: TBC sends this to verify the customer exists and get their info/balance.
  - Tests:
  ```javascript
  pm.test("Returns valid XML", function () {
      pm.expect(pm.response.headers.get("Content-Type")).to.include("xml");
  });

  pm.test("Contains result code 0 (success)", function () {
      var xml = pm.response.text();
      pm.expect(xml).to.include("<result>0</result>");
  });

  pm.test("Contains customer info fields", function () {
      var xml = pm.response.text();
      pm.expect(xml).to.include("<to_balance>");
  });
  ```

- `GET {{merchant_base_url}}/billing/?command=check&account=INVALID` — Invalid account
  - Tests: expects result code `5` (not found)

**PAY Command**
- `GET {{merchant_base_url}}/billing/?command=pay&txn_id={{tbc_txn_id}}&account={{billing_account}}&sum=50.00` — Process payment
  - Tests:
  ```javascript
  pm.test("Payment returns result 0", function () {
      var xml = pm.response.text();
      pm.expect(xml).to.include("<result>0</result>");
  });

  pm.test("Returns prv_txn (provider transaction ID)", function () {
      var xml = pm.response.text();
      pm.expect(xml).to.include("<prv_txn>");
  });
  ```

- `GET {{merchant_base_url}}/billing/?command=pay&txn_id={{tbc_txn_id}}&account={{billing_account}}&sum=50.00` — Duplicate payment (idempotency)
  - Description: Same txn_id sent twice. Should return code `215` (already processed).
  - Tests: expects result code `215`

### 4. OAuth2 Redirect — Merchant Endpoints (`tbc-oauth-merchant.postman_collection.json`)

For merchants using TBC Open Banking or TBC ID.

**OAuth2 Callback**
- `GET {{merchant_base_url}}/oauth/tbc/callback?code=test-auth-code&state=test-state` — OAuth2 redirect handler
  - Description: Where TBC redirects after user authorization. Tests that your handler processes the auth code.
  - Tests:
  ```javascript
  pm.test("OAuth callback processes successfully", function () {
      pm.expect(pm.response.code).to.be.oneOf([200, 302]);
  });
  ```

- `GET {{merchant_base_url}}/oauth/tbc/callback?error=access_denied&state=test-state` — OAuth2 error redirect
  - Description: Tests your error handling when user denies authorization.

## Test Scripts — Common Patterns

### Webhook handler health
```javascript
pm.test("Handler responds within TBC timeout window", function () {
    pm.expect(pm.response.responseTime).to.be.below(5000);
});

pm.test("Returns HTTP 200", function () {
    pm.expect(pm.response.code).to.equal(200);
});
```

### XML billing response structure
```javascript
pm.test("Valid XML billing response", function () {
    var xml = pm.response.text();
    pm.expect(xml).to.include("<?xml");
    pm.expect(xml).to.include("<result>");
    pm.expect(xml).to.include("<comment>");
});
```

### Redirect page verification
```javascript
pm.test("Page loads or redirects correctly", function () {
    pm.expect(pm.response.code).to.be.oneOf([200, 301, 302]);
});
```
